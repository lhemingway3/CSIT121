Web Security UI

How is it used?

When a site requires a certificate, these guidelines suggest that there should be a user interaction of clicking ‘accept’ or ‘decline’. Browsers should also show information about the certificate, like the name of the company and location. Accessibility is important here to make sure that the information is readable and the buttons are easy to navigate.

Users must be informed about where there information is going when they fill out a form. There must be a warning if there form submissions are going to an unsecured channel and that will be stopped from happening. If any HTTP transactions happen that are not strongly protected, the user should be notified.

Visual cues are important to convey any messages about security, but there needs to be multiple visual cues to be accessible for everyone. For example, using green and red will work for most people but not all. Symbols like lock icons are well understood to mean secure and will tell the user their information is safe.
